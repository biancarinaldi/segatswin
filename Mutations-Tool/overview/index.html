Overview

What is it?

Mutations is a tool used to examine the evolutionary aspects of a software system [Refer to Lehman's laws]

What does it do?

It extracts various metrics [link this to metric definitions] from a software system's source code and produces reports [link to report explanation pages] that help developers understand how their system is growing and changing [growing and changing needs further explanation].

How does it work?

It works by utilising the ASM [http://asm.ow2.org/] bytecode manipulation and analysis framework to extract metrics from Java bytecode [link to bytecode info page] belonging to a software system. This is performed upon the entire history [tooltip with explanation of 'history'] of the software system. Using these extracted metrics, Mutations forms an understanding of the software system and interprets how it is growing [define growth] and changing [define change] over its lifetime and what has brought has brought this about.

What is it good for?

  • Helps 'read between the lines' in terms of how a software system is evolving [be more clear]
  • Helps understand the tendencies of developers writing code
  • Articulates how a software system has been built for different audiences

What kind of reports does it generate?

  • Outlying versions report
  • Focus class report
  • Feature evolution report (coming soon)
  • Refactoring report (coming soon)
  • Architectural volatility report (coming soon)

What kind of statistics does it use?

  • Gini Coefficient
  • Theil Index
  • Chi-square Distance
  • Bhattacharyya Distance
  • Regression Analysis

What are it's broad aims?

  • Identification of concepts within systems (domain-level language, features, etc.)
  • Show how developers have built the system [be more clear]
  • Identify trends of growth/change for the system
  • Identify types of maintenance activities for the system
