User Manual

Installation and Configuration

Installation

To install Mutations, you must take the following steps:

  1. Create and set the MUTATIONS_HOME environment variable to the root directory for the Mutations source folder to the system path.
  2. Create and set the MUTATIONS_CONFIG environment variable to the Mutations.properties configuration file. If this file does not currently exist, create an empty file with that name.

Configuration

Next, you will need to set the configuration setting for the builds directory, which is where the systems you wish to run Mutations on are stored. To do this, change (or add if it does not exist) the following setting to your Mutations.properties file:

buildsDirectory=<Builds Directory>

In this case, <Builds Directory> will be replaced with the path to your builds directory, for example:

buildsDirectory=/Users/ajones/Development/builds

Running the Application

There are two ways of running Mutations, either using the Ant build script (build.xml) supplied with the source code, or using the metXF bash script.

Running with Ant build script

To run Mutations using the Ant build script, you must take the following steps:

  1. Navigate to the MUTATIONS_HOME directory
  2. Run Mutations using the command: ant run -Dargs="<args>", replacing <args> with the arguments you wish to pass to Mutations

Running with metXF bash script

To run Mutations using the metXF bash script, you must take the following steps:

  1. Navigate to the MUTATIONS_HOME directory
  2. Run the command ant package. This will compile Mutations into an a JAR file (under lib/Mutations.jar) that the metXF will use to execute Mutations.
  3. Ensure you have permissions to executre the metXF bash script, by entering the command chmod u+x metXF. This grants users execution priveleges for the metXF file.
  4. Run Mutations using the command ./metXF <System Name> <Arg1> ... <ArgN>, where <System Name> is the name of the system you wish to run Mutations on and <Arg1> ... <ArgN> are the arguments you wish to pass to Mutations.

Arguments

When executing Mutations, a total of 7 arguments can be passed to the application. The argument ordering is as follows:

<System Name> <Report Code> <Show Header> <Show Processing> <Report Arg 1> <Report Arg 2> <Report Arg 3>

Argument Definitions

The arguments that can be passed to Mutations are defined as:

System Name
The short name (first word) of the system to be analysed.
Report Code
The numeric code corresponding to the report to generate from the extracted data.
Show Header
A flag (Y/N) to indicate whether to show header on tabular reports being output.
Show Processing
A flag (Y/N) to indicate whether to show progress of processing occurring through the duration of execution.
Report Arg 1, 2, 3
Report specific arguments. See the Reports section for the list of arguments for each of the different reports.
