Overview

What is Helix?

Helix is a collection of open-source Java software systems that provides a means for analysis of software evolution. Each of the software systems that are apart of Helix are represented by their respective release history in the form of the compiled Java bytecode distributions which developers would normally acquire in order to use the systems.

What is the criteria for inclusion within Helix?

In order to be included within Helix, a software system must be:

  1. Developed for the Java virtual machine and code and compiled binaries for each release must be available
  2. A single, coherent system, i.e. it is a distribution of a collection of related components packaged together
  3. Beyond 15 available stable releases that have a version identifier (can be major or minor releases)
  4. Actively developed and used for at least 36 months
  5. Composed of at least 100 types (class and interfaces) in all releases under study
  6. Accompanied with change logs that provide information regarding major architectural shifts
